# Service Stack

### Service Order

Services start in dependency order (defined in `config/config.json`):

1. **trace** — Distributed tracing (gRPC, port 3001)
2. **vector** — Embedding vector store (gRPC, port 3002)
3. **graph** — RDF knowledge graph (gRPC, port 3003)
4. **pathway** — Standard data service (gRPC, port 3004)
5. **mcp** — MCP tool and prompt gateway (Streamable HTTP, port 3005)

### Environment

`npx fit-guide init` writes a `.env` file with service configuration:

| Variable            | Source                                |
| ------------------- | ------------------------------------- |
| `MCP_TOKEN`         | Generated by `init`                   |
| `SERVICE_*_URL`     | Written by `init`                     |
| `ANTHROPIC_API_KEY` | Set manually or via `fit-guide login` |

### MCP Authentication

All three surfaces (CLI, Claude Code, Claude Chat) authenticate to the MCP
endpoint with `Bearer <MCP_TOKEN>`. The CLI authenticates with Anthropic via
`ANTHROPIC_API_KEY` or OAuth (`fit-guide login`).
